Bug Submission Process

Fill out each field in the form including the priority.

How do I add an incident or bug issue?
To add bug tasks and receive notifications from JIRA you must have an account so please create your JIRA accounts! https://ilabdev.atlassian.net/
  1. Click on the Open the Bug Form button
  2. The Summary is the title.
  3. Please fill out all questions in the description field.
  4. Do not forget to include images or videos (you can save videos to google+ here).
  5. Do not forget to update the priority.
    1. It is very important that you select Urgent for urgent bugs or tasks.
  6. If someone else needs to be added as a watcher please indicate that in the description.
  7. Once the bug is submitted, Priority should only be updated by Product Management and Engineering Management.

Tracking Bugs and Bug Resolution

How do I track bugs?
  1. You will be added as a reporter to any bug you report.
  2. You will receive updates when the JIRA issue is updated.
  3. To track when a task should be completed, please reference the Due Date on the issue.
  4. Use Search for Issues to create a filter to track all bugs that need your attention.
  5. If an issue needs assistance from the reporter, another iLab team member or a third party it will be marked as On Hold with a comment as to what is needed to continue resolving the issue.
What is the bug resolutions workflow?
  1. If the issue is changed to User Acceptance Testing (UAT) then it is waiting for the Reporter to review the issue and confirm it is completed.
  2. The reporter should receive a notification letting them know which environment the issues was deployed to. (prd-dfci, prd-my, prd-zurich, demo or other test environments).
  3. Or the developer will leave a comment clearly letting the reporter know when and where they can confirm the issue is fixed.
  4. Once the reporter has confirmed the issue is fixed, then he/she can move the issue to DONE.
