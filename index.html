Bug Reporting Policies for Jira

Bug Submission Process

Fill out each field and attach files, if relevant.

How do I add an incident or bug issue?
To add bug tasks and receive notifications from Jira you must have an account so please create your Jira accounts! https://ilabdev.atlassian.net/
  1. Click on the Regular Bug Form button or the Urgent Bug Form
  2. The Summary is the title.
  3. Please fill out all questions in the description field.
  4. Do not forget to include images or videos (you can save videos to google+ here).
  5. If someone else needs to be added as a watcher please indicate that in the description.
  6. Once the bug is submitted, Priority will be evaluated by internal teams.

Tracking Bugs and Bug Resolution

How do I track bugs?
  1. You will be added as a reporter to any bug you report. Please remain logged into Jira for this to happen automatically.
  2. You will receive updates when the Jira issue is updated.
  3. Many factors are considered by Product Development when scheduling work on an issue.
  4. Use Search for Issues to create a filter to track all bugs that need your attention.
  5. If an issue needs assistance from the reporter, another iLab team member or a third party it will be marked as On Hold with a comment as to what is needed to continue resolving the issue.
What is the bug resolutions workflow?
  1. If the issue is changed to CLOSED & LIVE, you may want to check it yourself. If this incident has been resolved by Engineering adequately, please move the issue to LIVE & CONFIRMED. .
  2. Every issue goes through QA.
  3. If an issue is WAITING and you are the assignee, please complete the task or follow-up with the last assignee (i.e., R&D) for help.
